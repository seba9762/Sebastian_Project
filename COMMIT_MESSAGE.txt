fix: update analytics functions for schema changes (word_id, created_at)

This commit implements a comprehensive migration to fix all analytics SQL
functions to align with the updated database schema.

## Schema Changes Applied

All 13 analytics functions updated to use:
- learning_sessions.word_id (was: vocabulary_id)
- user_mistakes.word_id (was: vocabulary_id)
- user_mistakes.created_at (was: mistake_date)

Success logic now based on absence of user_mistakes records (not is_correct flag).
All date operations use Europe/Berlin timezone consistently.
